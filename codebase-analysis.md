# Codebase Analysis

## Repository shape

- `backend/`
  - FastAPI application
  - SQLAlchemy ORM
  - SQLite for tests, configurable database for runtime
  - workflow orchestration and local artifact storage
- `frontend/`
  - React 19
  - TypeScript
  - Vite
  - MUI component library
- `sample_data/`
  - scenario presets and image-risk examples

## Existing architecture patterns

### Backend

- Entry point: `backend/src/backend/main.py`
- Router composition: `backend/src/backend/api/routes.py`
- Persistence models: `backend/src/backend/models.py`
- API schemas: `backend/src/backend/schemas.py`
- Orchestration logic: `backend/src/backend/services/workflow.py`

Important observation:
- Each run already persists risk assessments, reports, events, artifacts, and policy decisions.
- That made aggregate analytics possible without schema changes.

### Frontend

- App shell and tab routing live in `frontend/src/App.tsx`
- API access lives in `frontend/src/api/client.ts`
- Shared types live in `frontend/src/types/api.ts`
- Page-level views live in `frontend/src/pages`

Important observation:
- The frontend already polls backend state and works well with a polling-based dashboard.
- The app uses simple tab state rather than URL routing.

## Existing user workflow

1. Upload a scenario from `Scenario Upload`
2. Wait for the run to complete
3. Inspect `Run Detail`
4. Generate `Hourly Reports`

## Gap found

- There was no portfolio-level view showing:
  - how risk accumulates across recent runs
  - where hotspots are clustering
  - which recommendations are repeating
  - which runs deserve immediate follow-up

## Why the chosen feature fit

- It reuses persisted data already generated by the workflow.
- It adds decision-making value without changing runtime behavior.
- It extends both backend and frontend in the same architectural style already present in the project.

## Integration points used

- Backend route added alongside existing `reports` and `runs` routes
- New backend service added under `services/`
- Frontend page added under `pages/`
- Frontend client/types extended rather than introducing a new state layer

## Risks considered

- Test state was previously shared across backend tests.
- Aggregate analytics can produce noisy results if the test database is not reset.
- Existing frontend text had encoding artifacts, so touched files were normalized to ASCII-safe separators.
